## Фаза 3 — Android Native: реальне блокування дзвінків

> Kotlin-сервіс, який перехоплює вихідні дзвінки на рівні Android OS.
> Це ключова частина MVP — без неї блокування є тільки в UI.

### Як це працює

```
Користувач набирає номер
↓
Android CallRedirectionService отримує подію
↓
Сервіс читає список заблокованих номерів з SharedPreferences
↓
Якщо номер у списку і захист активний:
  → дзвінок скасовується
  → додаток відкривається з поясненням
Якщо ні:
  → дзвінок дозволяється
```

### 5E-bis — Захист від видалення (Device Admin)

> Коли захист активний, користувач може обійти його просто видаливши додаток.
> Рішення: зробити додаток Device Administrator — тоді видалення заблоковане поки не знімеш роль адміна.
>
> Важливо: Device Admin вмикати ТІЛЬКИ поки захист активний, і знімати АВТОМАТИЧНО при вимиканні захисту.
> Тобто: увімкнув захист = додаток стає адміном. Вимкнув захист (через загадку) = адмін-роль знімається.
> Так це не "назавжди", а органічна частина режиму захисту.
>
> Реалізація:
> 1. `AdminReceiver.kt` extends `DeviceAdminReceiver` (порожня, базова достатня)
> 2. `res/xml/device_admin.xml` — порожні `<uses-policies>`
> 3. `AndroidManifest.xml` — зареєструвати receiver з `BIND_DEVICE_ADMIN`
> 4. `BlockedContactsModule.kt` — додати `requestDeviceAdmin()` та `isDeviceAdminActive()`
> 5. `BlockedContacts.ts` — прокинути нові методи
> 6. При вмиканні захисту (`setProtectionOn(true)`) → запросити Device Admin якщо ще немає
> 7. При вимиканні захисту (`setProtectionOn(false)`) → зняти Device Admin (`dpm.removeActiveAdmin`)

  1. В додатку на HomeScreen побачиш жовтий банер "Потрібен дозвіл" → натисни
  2. З'явиться системний діалог → обери "Ранок Розсудить"
  3. Банер зникне — роль надана

  ▎ Якщо діалог не з'явився або не показує твій додаток — перевір що телефон Android 10+. Можна перевірити вручну: Налаштування → Додатки → Стандартні → Перенаправлення дзвінків

  ---
  Крок 4 — Сценарій тестування

  4.1 Базовий тест — дзвінок блокується:
  1. Відкрий додаток → "Керувати контактами" → додай контакт з реальним номером (можна свій другий номер або будь-який)
  2. На HomeScreen увімкни захист (картка стає темно-зеленою)
  3. Вийди з додатку
  4. Відкрий стандартний телефон → набери той номер → натисни виклик
  5. Очікуваний результат: дзвінок миттєво скасовується, відкривається CallBlockedScreen з ім'ям контакту та індексом імпульсу

  4.2 Тест коли захист вимкнений:
  1. Вимкни захист на HomeScreen
  2. Знову спробуй набрати той самий номер
  3. Очікуваний результат: дзвінок проходить нормально

  4.3 Тест контакту без номера:
  1. Додай контакт без номера (тільки прізвисько)
  2. Набери будь-який номер
  3. Очікуваний результат: дзвінок не блокується (сервіс ігнорує контакти без номерів)

  Крок 5 — Дивитись логи якщо щось не працює

  C:\Users\nedik\AppData\Local\Android\Sdk\platform-tools\adb.exe logcat | Select-String "CallBlocker|CallRedirection|ranok|RANOK"
